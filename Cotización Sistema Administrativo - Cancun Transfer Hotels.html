1 Resumen Ejecutivo

Se propone el desarrollo del sistema administrativo completo para Mayan Transfer Experts. Esta propuesta complementa la cotización del sitio web público ($30,000 MXN) y cubre el panel de gestión desde el cual se administrarán reservas, clientes, vehículos, rutas, tarifas, contenido del sitio, pagos, comunicaciones (email y WhatsApp), CRM, reportes y futuras automatizaciones. El sistema contará con 24 módulos organizados en 7 fases de desarrollo.

5 Fases de Implementación

Fase 1 — Fundación y Autenticación Semanas 1–2
  • Setup del proyecto admin: React + Vite + TailwindCSS + TanStack
  • Extensión del backend: Prisma ORM (migración desde TypeORM), Docker Compose
  • Sistema de autenticación completo: login, logout, recuperar contraseña, JWT + refresh token
  • Roles y permisos: Admin y Operador (extensible a futuro)
  • Layout del panel: sidebar, header, navegación, tema
  • Guards de ruta y middleware de autorización
Fase 2 — Dashboard y Reservas Semanas 3–7
  • Dashboard: FullCalendar con vistas mensual, semanal y diaria
  • 7 indicadores KPI: nuevas, pendientes, confirmadas, próximas, actuales, finalizadas, canceladas
  • Acciones rápidas: nueva reserva, buscar, clientes, vehículos, calendario
  • CRUD completo de reservas: formulario multi-sección (cliente, viaje, pasajeros, vehículo, ubicaciones, vuelo)
  • Máquina de estados de reserva (7 estados con transiciones validadas)
  • Acciones: duplicar, confirmar, cancelar, reprogramar, imprimir, generar ticket PDF
  • Historial de actividad por reserva (log automático de cada acción)
  • Buscador global multi-entidad
Fase 3 — Catálogos Semanas 8–11
  • Viajeros / Clientes: CRUD + historial de reservas + métricas (viajes, gasto total)
  • Vehículos: CRUD con gestión de imágenes
  • Conductores: CRUD con asignación de vehículo
  • Tarifas: CRUD con dependencias múltiples (origen × destino × vehículo × temporada × moneda)
  • Destinos y Zonas: CRUD con imágenes
  • Hoteles: CRUD con coordenadas geográficas
  • Extras: CRUD (silla bebé, bebidas, decoraciones, paradas)
  • Cupones: CRUD con validación de fechas, límites y tipos de descuento
Fase 4 — CMS, Galería y Reportes Semanas 12–14
  • CMS: editor de contenido para 10+ secciones del sitio web
  • Administración de banner principal, botones, SEO (meta tags, Open Graph)
  • Galería: upload, edición, categorización y optimización automática de imágenes
  • Reportes: reservas diarias/mensuales, ingresos, cancelaciones
  • Reportes avanzados: vehículos más usados, hoteles top, clientes frecuentes
  • Exportación a Excel (ExcelJS) y PDF (PDFKit)
Fase 5 — Comunicaciones e Integraciones Semanas 15–18
  • Sistema de plantillas de correo: confirmación, pago, cancelación, recordatorio, post-viaje
  • Editor visual de plantillas con variables dinámicas ({nombre}, {folio}, {fecha})
  • Motor de automatizaciones: triggers configurables (crear reserva, confirmar pago, 24h/2h antes, post-viaje)
  • Integración WhatsApp Business API: envío de mensajes, confirmaciones, recordatorios
  • Compartir ticket PDF y ubicación por WhatsApp
  • Historial de comunicaciones por reserva
Fase 6 — CRM y Configuración Semanas 19–21
  • CRM: estados de cliente (Prospecto, Cliente, Frecuente, VIP)
  • Etiquetas, notas internas y seguimientos por cliente
  • Métricas: frecuencia de compra, total gastado, última reserva
  • Configuración general: datos de empresa, logo, redes sociales
  • Configuración de integraciones: Stripe, PayPal, SMTP, WhatsApp
  • Configuración de idiomas, monedas y tipo de cambio
Fase 7 — Seguridad, i18n y Deploy Semanas 22–24
  • Permisos granulares por módulo y acción
  • Auditoría: logs de actividad del sistema
  • Sistema de backups automáticos de base de datos
  • Internacionalización completa del panel (ES / EN)
  • Documentación API con Swagger / OpenAPI
  • Docker: contenedores para desarrollo y producción
  • Deploy en VPS Ubuntu + Nginx + PM2 + SSL
  • Pruebas integrales, QA responsivo (Desktop + Tablet) y entrega
